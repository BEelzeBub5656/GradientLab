## 一、项目概述与工具选型

### 网页项目主题
**校园智慧报修系统** — 一个面向高校后勤管理的单文件 Web 应用，实现学生报修提交与管理员工单处理的全流程闭环。

### 项目核心功能
1. **学生报修提交** — 填写故障类型、位置、描述、联系方式等信息，提交报修工单
2. **管理员工单处理** — 查看待处理工单列表，审核、分配维修人员、更新处理状态
3. **工单状态追踪** — 工单在「待审核→已分配→处理中→已完成」全生命周期流转，学生可查看进度
4. **数据本地持久化** — 使用浏览器 localStorage 存储所有工单数据，刷新不丢失
5. **角色切换** — 学生/管理员两种视角一键切换，体验完整的报修流程

## 六、Vibe Coding 学习总结与思考

### 1. 使用 AI 协作开发和自己手写代码的区别

Vibe Coding 改变了开发的思维方式。手写代码时，开发者需要同时关注业务逻辑、UI 设计、数据管理等多个层面，认知负荷大。而通过与 AI 对话驱动开发，可以将注意力集中在需求定义和方案决策上。以本次报修系统为例，工单状态流转的设计逻辑是人工确定的，但具体的代码实现（DOM 操作、事件绑定、localStorage 读写）均由 AI 完成。这种分工让人专注于"做什么"，AI 负责"怎么做"。

### 2. 好的提示词对最终成品的影响

本项目的开发经历了从模糊到精确的提示词演进。初始的"设计一个报修系统"仅能得到框架性代码；后续补充"学生端表单+管理端面板+状态流转+localStorage"等具体约束后，AI 的代码质量和完整度显著提升。好的提示词需要明确三点：功能边界、交互逻辑、技术约束。

### 3. 本次作业收获与后续改进思路

**收获**：掌握了单文件 HTML 应用从需求到上线的完整 Vibe Coding 流程，学会了利用 localStorage 实现无后端数据持久化，理解了角色切换的设计模式。

**后续改进**：可引入 IndexedDB 替代 localStorage 以支持更大数据量；可添加工单统计看板（数据可视化）；可接入真实的后端 API 实现多用户协同。

### 4. AI 代码生成存在的局限性，人在开发中不可替代的价值

AI 在代码生成上有明显局限：无法理解真实用户场景的细节（如报修表单哪些字段是必须的）、无法做出 UX 决策（如角色切换的交互方式）、倾向于生成模板化的 UI。人的价值在于：定义"做什么"（需求分析）、判断"好不好"（UX 评审）、确保"对不对"（逻辑验证）。
